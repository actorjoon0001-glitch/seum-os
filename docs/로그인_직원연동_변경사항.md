# 로그인 + employees 직원 연동 변경 사항

## 구현 목표 요약

- 이메일/비밀번호 로그인 후 **auth.users.id**로 **public.employees** 조회
- 직원 정보 없음 → 접근 제한 + 안내 메시지
- **status !== 'approved'** → 접근 제한
- 로그인 성공 + 직원 정보·승인 확인 후에만 대시보드 진입
- 대시보드 상단(사이드바)에 직원 이름 표시
- 직원 정보를 **window.seumAuth.currentEmployee**로 전역 저장 (팀/권한별 메뉴 분기 대비)

---

## 수정·추가된 파일과 변경 이유

### 1. `supabase/get_employee_by_auth_id.sql` (신규)

- **목적**: 로그인 사용자의 **auth_user_id**로 **public.employees** 한 건 조회
- **내용**: RPC `get_employee_by_auth_id(p_auth_user_id uuid)` 정의  
  - 반환: `id`, `name`, `team`, `role`, `showroom`, `status`, `permission`
- **이유**: Auth 로그인 후 한 번의 호출로 직원 정보를 가져와, 직원 없음/미승인 판단 및 전역 상태 저장에 사용

---

### 2. `js/auth.js` (수정)

| 변경 내용 | 이유 |
|-----------|------|
| **로그인 방식** | 사번+비밀번호 → **이메일+비밀번호**. Supabase Auth와 동일한 식별자 사용 |
| **fetchEmployeeByAuthId()** | auth_user_id로 employees 조회 로직을 한 곳으로 모아 로그인/requireAuth에서 재사용 |
| **fetchEmployeeByAuthIdFallback()** | `get_employee_by_auth_id` RPC 미적용 시 기존 `get_employee_status_by_auth_id`로 name/status만이라도 조회해 동작 유지 |
| **login()** | ① signInWithPassword(email, password) ② 성공 시 session.user.id로 직원 조회 ③ 직원 없으면 "직원 정보가 등록되지 않았습니다. 관리자에게 문의하세요." 후 signOut ④ status !== 'approved'면 "승인되지 않은 계정입니다. 관리자에게 문의하세요." 후 signOut ⑤ 직원 있음+승인 시 setCurrentEmployee() 후 대시보드로 이동 |
| **setCurrentEmployee()** | currentEmployee를 한 곳에서만 설정해 구조 통일 (id, name, team, role, showroom, status, permission, email) |
| **requireAuth()** | get_employee_by_auth_id(또는 fallback) 사용. 직원 없음 → `login.html?error=no_employee`, 미승인 → `login.html?error=not_approved` 로 리다이렉트 후 setCurrentEmployee 호출 |
| **window.seumAuth** | 초기화 시 `currentEmployee: null` 유지, 메서드만 할당해 기존 참조 깨지 않도록 함. Supabase 없을 때 console.error 제거(콘솔 에러 없이 동작) |

---

### 3. `login.html` (수정)

| 변경 내용 | 이유 |
|-----------|------|
| 입력 필드 | "사번" → **"이메일"** (type="email", id="login-email") |
| 제출 시 | `seumAuth.login(email, password)` 호출로 이메일 로그인과 직원 조회 연동 |
| 페이지 로드 시 | `?error=no_employee` → "직원 정보가 등록되지 않았습니다. 관리자에게 문의하세요." 표시, `?error=not_approved` → "승인되지 않은 계정입니다. 관리자에게 문의하세요." 표시 |
| seumAuth 호출 전 검사 | `window.seumAuth && window.seumAuth.login` 존재 여부 확인 후 호출해 스크립트 로드 순서/오류 시에도 콘솔 에러 방지 |

---

### 4. `dashboard.html` (변경 없음)

- 직원 이름 표시는 기존 **사이드바 하단 `#sidebar-current-user`** 사용
- `requireAuth()` 완료 후 `window.seumAuth.currentEmployee.name`으로 위 요소에 "OOO님" 설정되므로 추가 수정 없음

---

## 적용 순서 (Supabase)

1. **Supabase SQL Editor**에서 `supabase/get_employee_by_auth_id.sql` 전체 실행  
   → `get_employee_by_auth_id` RPC 생성  
2. (선택) 아직 없다면 `get_employee_status_by_auth_id`도 실행해 두면, RPC 미적용 시 fallback으로 name/status 조회 가능

---

## 전역 상태 사용 (팀/권한별 메뉴 분기 대비)

- **window.seumAuth.currentEmployee**  
  - `id`, `name`, `team`, `role`, `showroom`, `status`, `permission`, `email`  
- 로그인 성공·requireAuth 통과 후에만 채워짐  
- 추후 팀별/권한별 메뉴 노출은 `currentEmployee.team`, `currentEmployee.permission` 등으로 분기하면 됨
